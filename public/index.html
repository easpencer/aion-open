What Aion Does
1

Consolidate

Pull health data from every source — HealthKit, Health Connect, Withings, Dexcom, clinical records — into a single encrypted FHIR R4 store on your phone.

2

Understand

On-device AI models explore your data — medications, vitals, labs, conditions — and answer health questions without anything leaving your device.

3

Act

Agentic AI with tool calling. The AI can search the web, open links, set reminders, share content, and query your health data — with your permission at every step.

4

Bridge

When you want more compute, bridge your data to your PC over WiFi or USB. TLS-encrypted, audit-logged, with real-time subscriptions. Run Ollama, Claude Code, Python scripts — still your network, still your data.

How It Works

AI on your phone. Bigger AI on your PC.

Aion runs Edge Language Models directly on your device for private analysis. The local bridge lets your PC access the same data for deeper work with more powerful models.

Agentic AI

AI that can take action — with your permission

Aion's agent uses a ReAct loop (Reason → Act → Observe → Repeat) to answer complex health questions by calling tools. Every action goes through a tiered consent system.

🔍

Tool Calling

The AI can search the web, query your health data (FHIR), look up drug interactions, search medical evidence, open URLs, set reminders, and share content.

🔒

Tiered Consent

Auto — read-only tools run silently. Inform — shows what’s happening. Explicit — requires your tap to proceed. Prohibited — never runs.

🔑

Bring Your Own Key

Use your own API key for Claude, GPT-4, Gemini, or any OpenAI-compatible endpoint (Groq, Together, Ollama, Mistral, LM Studio). Keys never leave your device.

How the agent loop works
User: "What's my blood pressure trend and should I be concerned?"
                    ↓
    ① Agent calls query_vitals(days=30) → gets BP readings
    ② Agent calls compute_trend(coding="8480-6") → "trending up"
    ③ Agent calls search_health_evidence("hypertension management")
    ④ Agent synthesizes: "Your systolic BP has risen from 122 to 138
       over the past month. This is Stage 1 hypertension per ACC/AHA.
       Consider discussing with your provider at your next visit."

Recommended PC Models for the Bridge

When bridging to your PC, these models work well with Aion's tool-calling agent via Ollama:

Best Quality

Qwen 3 32B — Excellent tool calling, health reasoning. ~20GB VRAM. Llama 3.3 70B — Top-tier general reasoning. ~40GB VRAM (quantized). Mistral Large 2 — Strong multilingual + function calling. ~24GB.

Best Value (runs on most PCs)

Qwen 3 8B — Best 8B for tool calling. Fits 8GB VRAM. Recommended. Llama 3.1 8B — Solid all-rounder. 8GB VRAM. Gemma 3 4B — Runs on CPU. Good for laptops without GPU. Phi-4 14B — Microsoft's reasoning model. 10GB VRAM.
Install any model: ollama pull qwen3:8b — Aion auto-detects available models when you connect the bridge.

For Developers

Connect in a few lines

The bridge speaks WebSocket with JSON messages. Authenticate with the pairing code shown on your phone, then query any FHIR resource type.

Python
import asyncio, json, websockets

async def main():
    async with websockets.connect(
        "wss://your-phone:8420"
    ) as ws:
        await ws.send(json.dumps({
            "type": "auth",
            "payload": {"pairingCode": "123456"}
        }))
        auth = json.loads(await ws.recv())

        await ws.send(json.dumps({
            "type": "query",
            "payload": {
                "resourceType": "Observation",
                "category": "vital-signs"
            }
        }))
        data = json.loads(await ws.recv())
        for e in data["payload"]["entry"]:
            r = e["resource"]
            print(r["code"]["text"],
                  r.get("valueQuantity",{}).get("value"))

asyncio.run(main())
JavaScript
const ws = new WebSocket(
  "wss://your-phone:8420"
);

ws.onopen = () => {
  ws.send(JSON.stringify({
    type: "auth",
    payload: { pairingCode: "123456" }
  }));
};

ws.onmessage = (e) => {
  const msg = JSON.parse(e.data);

  if (msg.payload?.authenticated) {
    ws.send(JSON.stringify({
      type: "query",
      payload: {
        resourceType: "Condition"
      }
    }));
  }

  if (msg.payload?.entry) {
    for (const e of msg.payload.entry)
      console.log(e.resource.code?.text);
  }
};
