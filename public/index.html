How it works

Four ways to use Aion

No code needed

Just the App

Download Aion and use it standalone. Health dashboard, knowledge graph, and AI chat — entirely on your phone. Nothing ever leaves your device.

✓ HealthKit / Health Connect sync ✓ Vitals, conditions, medications ✓ Interactive knowledge graph ✓ On-device AI (works offline)

100% private. The bridge is there if you ever want to extend to your PC.

Local computing · Private

Bridge to Your PC

Run the bridge server on your PC and access your health data from any tool — browser, terminal, Jupyter, or custom code. Use local models with Ollama or connect any cloud provider. Still completely private.

✓ Ollama, Claude, OpenAI, Groq, Together.ai ✓ REST API for custom apps ✓ Terminal CLI + Jupyter notebooks ✓ Browser explorer (zero install)

Requires: Node.js 18+, phone and PC on same WiFi.

Claude Code & AI assistants

MCP: Ask Claude About Your Health

Give Claude Code direct access to your FHIR data via MCP. Ask “What’s my BP trend?” and get answers grounded in real data from your phone.

✓ Works with Claude Code, Claude Desktop ✓ Any MCP-compatible assistant ✓ Live FHIR data, not summaries ✓ Knowledge graph + analysis tools

Requires: Node.js 18+, phone and PC on same WiFi. Setup below.

Desktop Companion · Mac & Windows

Send Your PC to Your Graph

An Electron app that runs in your system tray and sends your computer activity to your health knowledge graph — anonymized, on-device. Your phone finally knows your whole story.

✓ Screen time, meeting hours, break count ✓ App categories (never specific names) ✓ Typing intensity: high / medium / low ✓ Correlates with BP, HRV, sleep, glucose

Build from source: cd aion-desktop && npm install && npm start

Setup guide

What runs where — the complete picture

Four independent pieces. Use only what you need. Each works without the others.

Step 1
Your Phone
Required for everything

Download the Aion app from the App Store (iOS) or Play Store (Android). It runs the bridge server automatically.

1. Open Settings → Local Bridge and turn it on 2. Note your phone's IP address (shown in the bridge screen) 3. Note the 6-digit pairing code (changes each session)
The bridge listens on port 8420. Your PC and phone must be on the same WiFi network.
Step 2 (optional)
Bridge Server
REST API + browser console

Runs aion-open on your PC. Exposes your phone's FHIR data as a standard REST API at localhost:3000. Includes a browser console, CLI, and code generator.

Requires: Node.js 18+
# One-time setup git clone https://github.com/easpencer/aion-open cd aion-open && npm install # Run (replace with your phone's IP and code) BRIDGE_URL=wss://192.168.1.42:8420 PAIRING_CODE=123456 npm start

Then open http://localhost:3000 in any browser. No login, no cloud.

Step 3 (optional)
MCP Server
Claude ↔ your health data

A separate Node process that connects Claude Code or Claude Desktop directly to your phone's bridge. Gives Claude tools to query FHIR data, run analysis, read your knowledge graph, and write new health observations back to your phone — all live, from real data.

Requires: Node.js 18+. Connects directly to your phone — the bridge server (Step 2) is not required.
# One-time setup git clone https://github.com/easpencer/Aion cd Aion/aion-web/src/services/bridge/mcp npm install

Add to your .mcp.json (Claude Code: ~/.mcp.json or project root):

{ "mcpServers": { "aion-health": { "command": "node", "args": ["/absolute/path/to/mcp/dist/index.js"], "env": { "AION_BRIDGE_URL": "wss://192.168.1.42:8420", "AION_PAIRING_CODE": "123456" } } } }

Restart Claude Code. Then ask: “What are my BP trends?” or “What medications am I on?”

Step 4 (optional)
Desktop Companion
PC activity → health graph

An Electron app for Mac/Windows that captures your computer activity and sends it to your health knowledge graph — anonymized, categories only, no content. Your graph discovers correlations like “BP higher on weeks with >6h meetings.”

✓ Screen time minutes ✓ Meeting hours per day ✓ Break count + sedentary streaks
✓ App categories (productivity, communication…) ✓ Typing intensity: high / medium / low ✗ Never: URLs, app names, content, keystrokes
macOS arm64 .dmg available now (unsigned dev build). Windows .exe coming soon.
# Run from source cd aion-desktop && npm install && npm start

Enter your phone’s IP and pairing code in the tray app. It syncs hourly in the background.

REST API

Endpoints

All return JSON. Connect first, then query freely.

METHODENDPOINTDESCRIPTION
GET/api/{type}Query FHIR resources. Params: category, limit, start, end
GET/api/{type}/{id}Get a single resource by ID
POST/api/analyzeAsk a health question. Body: {"question":"..."}
GET/api/_exportExport FHIR Bundle or CSV
GET/api/_metadataFHIR R4 CapabilityStatement
GET/api/_graphKnowledge graph: nodes, edges, communities, god nodes
GET/api/_graph/reportHEALTH_REPORT.md — structured summary for LLM context
GET/api/_auditView bridge audit log
POST/api/connectConnect to phone bridge
GET/api/statusConnection status
Code Examples

Build with any language

Standard HTTP. No SDK needed.

Python
import requests

resp = requests.get(
    "http://localhost:3000/api/Observation",
    params={"category": "vital-signs", "limit": 20}
)
for entry in resp.json().get("entry", []):
    r = entry["resource"]
    name = r.get("code", {}).get("text", "?")
    val = r.get("valueQuantity", {})
    print(f"  {name}: {val.get('value')} {val.get('unit', '')}")

# Ask your AI about your health
resp = requests.post(
    "http://localhost:3000/api/analyze",
    json={"question": "Summarize my recent vitals"}
)
print(resp.json().get("answer", ""))
JavaScript
const resp = await fetch(
  "http://localhost:3000/api/Observation?category=vital-signs"
);
const data = await resp.json();

for (const { resource } of data.entry || []) {
  console.log(
    `${resource.code?.text}: ${resource.valueQuantity?.value} ${resource.valueQuantity?.unit}`
  );
}

// Ask your AI about your health
const analysis = await fetch("http://localhost:3000/api/analyze", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ question: "Any concerns with my BP?" })
});
console.log((await analysis.json()).answer);

Zero Mile Health

  
Your data. Your control. Your innovation.

  
Your health data has always been yours. The problem is no one lets you use it. It sits in portals you can barely access, locked in formats you can’t read, controlled by systems that don’t answer to you. Aion changes that. Your data lives on your device, encrypted, private, and queryable — a FHIR knowledge graph that you own completely. On-device AI gives you intelligence that works offline. Bridge to your PC for deeper analysis. Go online when you choose, with your own models, on your terms.

  
But the real power isn’t just access — it’s what you build with it. A researcher could correlate medication adherence with sleep patterns. A developer could build a health dashboard for their family. A clinician could create a patient-facing risk visualization. A student could train a model on their own vitals. We don’t know what you’ll build. That’s the point.

  
We believe essential healthcare should be universal — meeting people where they are, not where systems demand they go. Aion is the start of something different. A digital twin that knows your complete story. AI that reasons about you — first on your device, then on your network, then with any model you trust. Health at zero miles — in your pocket, understanding you better every day.

  
This is early. This is open. Build something amazing.
